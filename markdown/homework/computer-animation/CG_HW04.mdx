## 作品技術概念

用 Geometry Node 製作政大 LOGO 的 Procedural LOGO Sequence Animation。
首先，先將上週製作的 Image Sequence 導入 Geometry Nodes，並利用本週所習得之 Instance on Point 的概念，套疊至課堂所使用的 Animation node，再以 Geometry Proximity Node 搭配 slicePlane 進行 Extrusion instance 之控制。 再將 所有模型物件相關之 Geometry Nodes 集合在一個 NodeGroup 中，最後增加可調式參數控制區。
Loop Animation 時長為 20 秒鐘 (F=480)，以 keyframe 去控制每個物件的動畫細節。

- 為了讓 instance 可以由底部開始上下伸縮（意思是物體中心需在底部），需將物體往上移動。因為需要移動 instance 深度的一半，所以我將 depth 變成可調式參數，並將此參數除以 2、接到 combine XYZ 的 Z 方向上，就可以調整 instance 的位置。
3. 匯入上週做的 logo animation 的 image sequence（原本不知道如何加入多張圖片，詢問 ai 後發現只要一次匯入所有圖片即可）。
接者拆 uv，就是**將 3D 物體的表面「攤平」成 2D 平面的過程** ，好讓 2D 的圖片能精準地貼在這塊 Grid 上面。
因為我的 image sequence 只有 120 張，這次作業要做 10 秒（frame=240）以上（我做了 20 秒），所以須調整一下圖片出現的時間。
- 首先，透過 scene time 的 frame 接到 image texture 的 frame 上，可以控制第 1 張圖片出現再frame=1、 第 2 張圖片出現再frame=2，以此類推。
- 接者加入 math 節點的 ping-pong，意思是讓 image texture 可以**正常播完再倒著** **播**（scale=119 是前 119 張圖片要倒著播的意思）。
- 因為我要做 20 秒（frame=480，因為 10 秒動畫會看起來太快），目前是 frame=240，所以我要將每張圖片連續出現兩個 frame（也就是 frame=1、frame=2 都出現第一張圖）。透過將 frame 乘以 0.5 可以達成（也可以想成 0.5 倍速播放）。
- 最後將 image texture 的 color 接在 instance on point 節點的 selection 上，意思是將圖片與剛才的 instance 對應，有顏色的部分才讓 instance 顯現出來。

1. 首先我先讓 slicePlane 動起來，才能讓 logo 上的 instance 能夠自動有高低變化。
- 這次不用 keyframe 的方式，而是用 **Driver。**Driver 是一種用函數或 數學表達式 來控制屬性值的方法。它可以用 python 來完成，不用手動加上每個 keyframe，非常方便。
- 因為我想讓 slicePlane 來回動，且動的幅度大一些、速度慢一些，所以我將 Driver 的值設定成 # sin(frame * 0.04)*2 。sin(x) 的 x 越小，函數的週期越大，等同於走完一個波的時間越長，也就是速度越慢。sin(x)*k 的 k 越大，代表波的振幅越大，也就是來回的距離越遠、動的幅度越大。
- 我不希望 slicePlane 全程影響 instance 高度，所以需要有範圍限制，因為可以用單行 python 來寫，所以完整的 driver 會是：# sin(frame * 0.04)*2 if (40 &lt;= frame &lt;= 200 or 280 &lt;= frame &lt;= 440) else 2，如果 frame 介於 40~200 或 280~440， slicePlane 就會動，否則就固定在與 logo 距離 1 公尺（因為有 map range 映射數值到 1），這樣讓動畫比較不會太重複、太枯燥。
2. 在 logo 上我也設定一些 keyframe，像是調整 Instance 的整體高度、半徑，中間讓 instance 沿著 x 與 y 方向旋轉等，如圖2

## 自我評量與作品建議

**9/10**

這次製作過程還算順利，沒有遇到什麼太大的問題，可能是因為大部分的 node 都使用上課做的內容，只有自己加入少許的節點來完成作品。
作品其實一開始只有設定 10 秒，但後來覺得動畫動得太快，所以將所有 keyframe 都拉長兩倍（將時間軸放在第一個 keyframe 上，按 【S】【2】 就可以放大兩倍，非常方便 ），而 image sequence 也要變慢，我思考後就想到可以透過將輸入的 frame 數乘以 0.5，讓 frame=2 也可以跑第一張圖、 frame=4 也可以跑第二張圖，嘗試後發現真的是這樣，然後就順利幫動畫減速了。
我覺得 geometry node 很讚，因為它的非破換性，讓我在後來想要修改前面的東西時，可以很方便的做更改。
**過程中遇到的困難：**
### step 1-1
- 一開始不知道要怎麼把很多張的 image sequence 加進 Image texture 中 ，詢問 ai 後發現只要一次全選並匯入所有圖片即可。
- 我想製作圖片順著播再倒著播的感覺，詢問 ai 後學到可以用 math 節點的 ping-pong 來完成，它會 限制 輸入的數值在 0 與指定 scale 之間來回循環。
